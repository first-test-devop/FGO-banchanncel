# Chaldea Bond

《Fate/Grand Order》阵容羁绊礼装分析工具。用户选择五名自有英灵与一名助战后，系统会根据关卡基础羁绊、助战礼装规则和可用礼装库存，计算整队总羁绊最高的装备方案，并解释每个推荐。

> 当前版本：`v0.3.0`（2026-06-24）。这是可运行的 Web MVP，游戏数据与规则可能随服务器版本更新，请以游戏内说明为准。

## v0.3.0 版本更新

相较 `v0.2.0`，本版本重点把“推荐得出来”推进到“玩家实际带得上”，并覆盖冠位战的多礼装位场景：

- 新增编队 Cost 上限输入，并以五名自有从者 + 自有礼装的实际 Cost 为约束寻找最优方案。
- 补齐国服从者与羁绊礼装 Cost，分析结果展示已用 Cost、剩余 Cost 与礼装成本。
- 助战从者的礼装改为选择助战时固定，可选择未满破 / 满破，也可以为空；分析器不再替换助战礼装。
- 新增“普通关卡 / 冠位战”模式。冠位战必须指定一名自有冠位英灵，助战也可标记为冠位。
- 冠位英灵展示三个礼装位：位 1 可优化且正常计 Cost，位 2 为固定自身羁绊礼装，位 3 可优化且 Cost 视为 0。
- 冠位助战的可选礼装位由用户固定，允许留空，并且不计入玩家编队 Cost。
- Cost 不足时会输出空礼装位，而不是推荐游戏内无法实际编成的方案。
- 自动化测试扩展到 `18` 项，覆盖 Cost、助战空礼装与冠位战多礼装位规则。

完整版本记录见 [CHANGELOG.md](./CHANGELOG.md)。

## 核心能力

- 六槽位阵容编成：五名自有英灵 + 一名助战
- 支持输入编队 Cost 上限，按五名自有从者与自有礼装的精确 Cost 寻找可执行方案
- 选择助战英灵时同时固定其礼装和突破状态，也可选择不携带礼装；助战配置不占玩家编队 Cost，也不会被分析器替换
- 支持普通关卡与冠位战双模式；冠位战必须指定一名自有冠位英灵，助战可选择是否为冠位助战
- 冠位英灵的礼装位 1 正常计算 Cost，固定羁绊礼装位 2 与报酬提示礼装位 3 均按 Cost 0 处理
- 冠位英灵的礼装位 1、位 3 都允许留空并参与羁绊最优组合；冠位助战的两个可选槽由用户固定
- 已选择卡槽支持按住拖拽并与目标槽直接交换，助战身份随卡槽移动
- 441 名国服英灵的名称、职阶和图鉴编号搜索
- 通过搜索建立参与分析的羁绊礼装池，再为已选礼装设置未满破或满破
- 礼装池与突破状态保存在本机浏览器中，刷新或再次打开后无需重复录入
- 国服常驻羁绊礼装完整池，包含特性限定 `+20%` 与 FSN 从者 `+10%` 礼装
- 八周年常驻首发成员羁绊 `+20%` 规则
- 百分比与固定值收益的精确比较
- 助战位「迦勒底午茶时光」15% 特殊规则
- 助战可与任意槽位换位，并计算前排助战触发的全队 `+4%`
- 每槽位推荐、收益解释和整队汇总
- 每名英灵展示礼装乘区、站位乘区和固定羁绊三步算式
- 英灵选择、阵容与分析结果展示羁绊计算相关特性，并解释特性礼装的实际受益对象
- 响应式布局，支持手机与桌面浏览器

## 技术架构

```text
src/
├── components/       # 纯 UI 组件
├── data/             # 版本化的英灵与礼装数据
├── domain/           # 与框架无关的羁绊规则、优化器和类型
└── lib/              # 日志等横切能力
```

技术栈为 React 19、TypeScript、Vite、Vitest 与 ESLint。领域层不依赖 React，可在后续微信小程序项目中直接复用，或独立发布为共享 npm workspace package。

优化器当前穷举 Cost 上限内最多五张自有礼装的组合。阵容规模与礼装池有限，这种实现比贪心策略更易审计，并可正确处理空礼装位、助战固定礼装、特性限定效果与固定加成。

## 部署

当前 Beta 版部署在 Cloudflare Pages 免费项目中。`main` 分支推送后会触发 `.github/workflows/release.yml`：先执行 `pnpm lint`、`pnpm test`、`pnpm build`，通过后将 `dist` 发布到 Cloudflare Pages。

GitHub Actions 自动发布需要配置 `CLOUDFLARE_ACCOUNT_ID` 与 `CLOUDFLARE_API_TOKEN` 两个仓库 Secret。详细配置见 [docs/deployment.md](./docs/deployment.md)。

本地直传部署：

```bash
pnpm build
pnpm dlx wrangler pages deploy dist --project-name chaldea-bond-beta --branch main
```

## 数据更新

英灵基础数据来源于 [Atlas Academy](https://atlasacademy.io/) 的公开游戏数据导出。下载国服 `basic_servant.json` 与 `nice_servant.json` 后执行：

```bash
node scripts/generate-servants.mjs \
  /path/to/basic_servant.json \
  src/data/servants.json \
  /path/to/nice_servant.json
```

生成器只保留产品需要的字段，避免把上游大型数据文件直接打包进前端。

长期数据维护采用“自动检查 + 人工确认 + 版本发布”流程：

- `pnpm data:check` 会拉取 Atlas Academy 国服导出，生成从者差异与疑似羁绊礼装候选报告。
- `.github/workflows/data-freshness.yml` 每周自动运行一次数据检查，并上传报告。
- 新增羁绊礼装不会自动进入正式计算，必须确认效果、数值、目标特性、助战特殊倍率与 Cost 后再入库。
- 页面会展示当前数据版本，版本信息维护在 `src/data/dataManifest.ts`。

详细流程见 [docs/data-maintenance.md](./docs/data-maintenance.md)。

## 当前计算边界

- 按常驻关卡结算规则计算，前三个首发槽位中的自有英灵额外获得 20% 羁绊。
- 基础公式为：
  1. `floor(基础羁绊 × (1 + 礼装百分比 + 活动百分比))`
  2. 对第一步结果应用站位乘区并再次向下取整
  3. 最后加入英灵肖像等固定羁绊
- 助战在首发时，其首发 20% 平摊为五名自有英灵各 4%；首发自有英灵因此使用 24%，后备使用 4%。
- 活动百分比字段当前保留为 0%，尚未接入活动规则。
- 「来自ＮＦＦ的爱」「检查报告」「至诚的一针」等特性礼装只对满足条件的英灵生效，分析器会根据英灵职阶与特性逐名计算。
- “拥有灵衣之人”是系统固定特性；即使玩家账号尚未取得灵衣开放权、尚未开放或当前未穿着，也符合「至诚的一针」的条件。
- 同名羁绊礼装当前按最多使用一张计算，效果采用用户选择的未满破或满破状态。
- 编队 Cost 只统计五名自有从者和分配给他们的自有礼装；助战从者及助战礼装不计入 Cost。
- 助战礼装由用户在选择助战时固定；分析器仅优化自有礼装，Cost 不足时允许部分自有从者不装备羁绊礼装。
- 普通关卡中冠位英灵仍按普通英灵处理，只有一个礼装位。
- 冠位战中必须且只能指定一名自有冠位英灵；其礼装位 2 固定为自身羁绊礼装，不参与羁绊加成优化。
- 冠位英灵礼装位 1 的礼装正常计算 Cost；礼装位 2 和礼装位 3 的礼装 Cost 均视为 0。
- 助战英灵不计入玩家获得的羁绊，但其礼装对自有成员生效。
- 玛修按无法通过普通关卡获得羁绊处理，其礼装仍能为全队加成。
- 暂未纳入活动或主线期间按职阶、性别、阵营等特性提供的限时羁绊加成。
