# Chaldea Bond

《Fate/Grand Order》阵容羁绊礼装分析工具。用户选择五名自有英灵与一名助战后，系统会根据关卡基础羁绊、助战礼装规则和可用礼装库存，计算整队总羁绊最高的装备方案，并解释每个推荐。

> 当前版本：`v0.2.0`（2026-06-23）。这是可运行的 Web MVP，游戏数据与规则可能随服务器版本更新，请以游戏内说明为准。

## v0.2.0 版本更新

相较远程仓库上一版本，本版本重点改善了礼装库存录入和特性限定礼装的可解释性：

- 羁绊礼装支持“未满破 / 满破”，分析器按实际突破状态使用对应数值。
- 礼装库存改为搜索、多选后再设置突破状态，不再要求用户逐项处理整张礼装表。
- 礼装池与突破状态保存在本机浏览器，刷新或再次打开后无需重复录入。
- 英灵选择、阵容卡片与分析结果展示羁绊计算相关的职阶、阵营、性别等特性。
- 特性限定礼装明确列出实际受益英灵和命中条件，并说明佩戴者本人无需符合该特性。
- 无匹配对象的特性礼装标记为“补位礼装 · 当前无特性收益”，避免误导。
- 明确“拥有灵衣之人”属于系统固定特性，不受玩家是否取得灵衣开放权、是否开放或是否穿着影响。
- 新增相关领域测试，当前共 `12` 项自动化测试。

完整版本记录见 [CHANGELOG.md](./CHANGELOG.md)。

## 核心能力

- 六槽位阵容编成：五名自有英灵 + 一名助战
- 支持输入编队 Cost 上限，按五名自有从者与自有礼装的精确 Cost 寻找可执行方案
- 选择助战英灵时同时固定其礼装和突破状态，也可选择不携带礼装；助战配置不占玩家编队 Cost，也不会被分析器替换
- 已选择卡槽支持按住拖拽并与目标槽直接交换，助战身份随卡槽移动
- 441 名国服英灵的名称、职阶和图鉴编号搜索
- 通过搜索建立参与分析的羁绊礼装池，再为已选礼装设置未满破或满破
- 礼装池与突破状态保存在本机浏览器中，刷新或再次打开后无需重复录入
- 国服常驻羁绊礼装完整池，包含特性限定 `+20%` 与 FSN 从者 `+10%` 礼装
- 八周年常驻首发成员羁绊 `+20%` 规则
- 百分比与固定值收益的精确比较
- 助战位「迦勒底午茶时光」15% 特殊规则
- 助战可与任意槽位换位，并计算前排助战触发的全队 `+4%`
- 每槽位推荐、收益解释和整队汇总
- 每名英灵展示礼装乘区、站位乘区和固定羁绊三步算式
- 英灵选择、阵容与分析结果展示羁绊计算相关特性，并解释特性礼装的实际受益对象
- 响应式布局，支持手机与桌面浏览器

## 技术架构

```text
src/
├── components/       # 纯 UI 组件
├── data/             # 版本化的英灵与礼装数据
├── domain/           # 与框架无关的羁绊规则、优化器和类型
└── lib/              # 日志等横切能力
```

技术栈为 React 19、TypeScript、Vite、Vitest 与 ESLint。领域层不依赖 React，可在后续微信小程序项目中直接复用，或独立发布为共享 npm workspace package。

优化器当前穷举 Cost 上限内最多五张自有礼装的组合。阵容规模与礼装池有限，这种实现比贪心策略更易审计，并可正确处理空礼装位、助战固定礼装、特性限定效果与固定加成。

## 数据更新

英灵基础数据来源于 [Atlas Academy](https://atlasacademy.io/) 的公开游戏数据导出。下载国服 `basic_servant.json` 与 `nice_servant.json` 后执行：

```bash
node scripts/generate-servants.mjs \
  /path/to/basic_servant.json \
  src/data/servants.json \
  /path/to/nice_servant.json
```

生成器只保留产品需要的字段，避免把上游大型数据文件直接打包进前端。

## 当前计算边界

- 按常驻关卡结算规则计算，前三个首发槽位中的自有英灵额外获得 20% 羁绊。
- 基础公式为：
  1. `floor(基础羁绊 × (1 + 礼装百分比 + 活动百分比))`
  2. 对第一步结果应用站位乘区并再次向下取整
  3. 最后加入英灵肖像等固定羁绊
- 助战在首发时，其首发 20% 平摊为五名自有英灵各 4%；首发自有英灵因此使用 24%，后备使用 4%。
- 活动百分比字段当前保留为 0%，尚未接入活动规则。
- 「来自ＮＦＦ的爱」「检查报告」「至诚的一针」等特性礼装只对满足条件的英灵生效，分析器会根据英灵职阶与特性逐名计算。
- “拥有灵衣之人”是系统固定特性；即使玩家账号尚未取得灵衣开放权、尚未开放或当前未穿着，也符合「至诚的一针」的条件。
- 同名羁绊礼装当前按最多使用一张计算，效果采用用户选择的未满破或满破状态。
- 编队 Cost 只统计五名自有从者和分配给他们的自有礼装；助战从者及助战礼装不计入 Cost。
- 助战礼装由用户在选择助战时固定；分析器仅优化自有礼装，Cost 不足时允许部分自有从者不装备羁绊礼装。
- 助战英灵不计入玩家获得的羁绊，但其礼装对自有成员生效。
- 玛修按无法通过普通关卡获得羁绊处理，其礼装仍能为全队加成。
- 暂未纳入活动或主线期间按职阶、性别、阵营等特性提供的限时羁绊加成。
