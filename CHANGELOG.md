# Changelog

本项目遵循 [Semantic Versioning](https://semver.org/)；日期格式为 `YYYY-MM-DD`。

## [Unreleased]

### Added

- 新增“普通关卡 / 冠位战”模式切换。
- 冠位战支持指定一名自有冠位英灵，以及可选的冠位助战。
- 自有冠位与冠位助战均展示三槽结构：可配置礼装位 1、固定自身羁绊礼装位 2、可配置报酬提示礼装位 3。
- 支持输入玩家当前编队 Cost 上限，并展示从者 Cost、礼装 Cost、已用与剩余 Cost。
- 从国服游戏数据补齐 441 名从者和羁绊礼装的精确 Cost。
- 选择助战英灵后继续选择其固定礼装与突破状态，并支持助战不携带礼装。

### Changed

- 冠位战优化器最多处理六个自有可配置礼装位；位 1 正常计算 Cost，位 3 按 Cost 0 参与优化。
- 冠位助战的位 1、位 3 由用户固定选择，均允许留空且不计玩家 Cost。
- 优化器只为五名自有从者分配礼装，并在 Cost 上限内最大化整队羁绊。
- Cost 不足时允许输出空礼装位，而不是推荐无法实际编成的方案。
- 助战从者和助战礼装不计入玩家 Cost，助战礼装不再由优化器替换。

## [0.2.0] - 2026-06-23

### Added

- 羁绊礼装的未满破与满破状态建模，并按对应数值参与优化计算。
- 支持搜索、批量选择本次纳入分析的羁绊礼装。
- 使用浏览器本地存储保存礼装池、突破状态和基础羁绊设置。
- 在英灵选择、阵容卡片和分析结果中展示羁绊计算相关特性。
- 在特性限定礼装结果中展示实际受益英灵及命中的具体条件。
- 新增英灵特性解释模块与自动化测试。

### Changed

- 礼装库存交互从“逐项设置整张列表”改为“先选择礼装，再设置突破状态”。
- 分析结果会区分有效的特性礼装和无匹配对象的补位礼装。
- “至诚的一针”的目标说明统一为“拥有灵衣之人（系统固定特性）”。
- README 增加当前版本、核心变化和计算边界说明。

### Fixed

- 修正未满破羁绊礼装此前无法纳入分析的问题。
- 修正特性限定礼装缺少受益对象解释、容易被误解为仅对佩戴者生效的问题。
- 修正“拥有灵衣之人”可能被误解为需要玩家已取得灵衣开放权的问题。

### Validation

- ESLint 检查通过。
- Vitest：3 个测试文件、12 项测试通过。
- TypeScript 与 Vite 生产构建通过。

## [0.1.0] - 2026-06-22

### Added

- 六槽位阵容编成、英灵搜索与助战位置调整。
- 卡槽拖拽换位。
- 常驻羁绊礼装池和最优装备分析。
- 三步羁绊计算过程与推荐理由展示。
